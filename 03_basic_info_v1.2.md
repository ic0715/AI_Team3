# 03. 기본 정보 입력

> AI 코칭 개인화를 위해 이름·나이대·직군·현재 고민 등 기본 프로필을 수집함. 이후 강점·커리어 분석의 컨텍스트로 활용됨.

## 1. 화면 개요

| 항목 | 내용 |
| --- | --- |
| 화면 ID | 03_basic_info |
| 페이즈 | ONBOARDING |
| 역할 | 닉네임/생년월일/직업/경력/고민 등 기본 프로필 수집 |
| 이전 화면 | 02 로그인/회원가입 (또는 NEW01 이메일 인증 완료 후) |
| 다음 화면 | 04 강점 진단 방식 선택 (신규) / 15 프로필 또는 NEW03 (수정 모드) |

## 2. 진입 조건

- **신규 진입 (온보딩)**: NEW01 이메일 인증 완료 후, `profiles.profile_completed = false` (또는 NULL)일 때 자동 진입
- **수정 모드 (재진입)**:
  - 15 프로필 → [정보 수정] 버튼
  - NEW03 12주 완료 → [기본 정보 업데이트] 보조 버튼
  - 수정 모드에서는 기존 값을 prefill, 완료 후 진입 화면으로 복귀

> ⚠️ **schema 불일치 수정**: 기존 `users.basic_info_completed_at` 컬럼은 schema에 존재하지 않음.  
> 완료 여부는 `profiles.profile_completed (boolean)` 컬럼으로 대체.  
> → **schema 반영 필요 항목**: `profiles` 테이블에 `profile_completed boolean DEFAULT false` 컬럼 추가

## 3. UI 구성

### 3.1 상단 바

- 뒤로가기 버튼: 신규 진입 시 비활성(또는 미노출), 수정 모드(15 또는 NEW03에서 진입) 시 진입 화면으로 복귀
- 페이지 타이틀: 신규 진입 = "기본 정보", 수정 모드 = "기본 정보 수정"

### 3.2 인트로 메시지

- **신규 진입**: "반가워요! 👋 간단히 소개해주세요" + "맞춤 분석을 위해 필요한 정보예요. 언제든 수정할 수 있어요"
- **수정 모드**: "기본 정보를 업데이트해주세요" + "변경 사항은 다음 AI 인터뷰부터 반영돼요"

### 3.3 폼 필드

| 필드 | 타입 | 필수/선택 | 수정 모드 | 검증 |
| --- | --- | --- | --- | --- |
| 닉네임 | text | 필수 | 수정 가능 | 1~10자, 한글/영문/숫자, 욕설 필터, 중복 검사 |
| 생년월일 | date picker | 필수 | 🔒 수정 불가 (잠금) | 만 14세 이상, 미래 날짜 불가 |
| 성별 | 버튼 그룹 | 선택 | 수정 가능 | 남성/여성/기타/응답 안 함 |
| 현재 직업/분야 | dropdown | 필수 | 수정 가능 | 사전 정의 카테고리 |
| 경력 | dropdown | 필수 | 수정 가능 | 신입/1~3년/4~7년/8~12년/13년+ |
| 현재 커리어 고민 | textarea | 선택 | 수정 가능 | 0~200자, 글자수 표시 |

> 수정 모드에서 **생년월일은 잠금** 처리 (만 14세 이상 검증을 가입 시 통과한 정보 보존). 변경이 필요한 경우 고객센터 문의 안내.

### 3.4 직업/분야 카테고리

카테고리 목록은 클라이언트 상수로 관리. 초기 목록:

> ⚠️ **schema 불일치 수정**: `job_categories` 테이블은 schema에 존재하지 않음.  
> `profiles.job_field (text)`에 선택한 카테고리 텍스트를 저장하는 방식으로 운영.

- IT/개발, 디자인, 기획/PM, 마케팅/영업, HR/인사
- 재무/회계, 교육, 의료/보건, 제조/생산
- 학생, 기타 (직접 입력)

### 3.5 Bottom CTA

- **신규 진입**: "다음으로 →" - 필수값 모두 충족 시에만 활성화 → 04 강점 진단 방식 선택으로
- **수정 모드**: "저장" - 변경된 필드가 1개 이상이고 필수값 충족 시 활성화 → 변경된 필드만 UPDATE → 진입 화면(15 또는 NEW03)으로 복귀 + 토스트 "정보가 수정되었어요"
- **수정 모드 보조**: "취소" - 변경사항 있으면 다이얼로그 ("저장하지 않고 나가시겠어요?") → 진입 화면으로 복귀

| 기능 | 동작 | 비고 |

| 닉네임 중복 검사 | 입력 후 0.5초 디바운스 → 서버 확인 | 중복 시 인라인 에러 |

| 저장 클릭 (수정 모드) | 클라이언트 검증 → 변경된 필드만 UPDATE → 진입 화면 복귀 + 토스트 | 저장 중 로딩 표시 |
| 뒤로가기 / 취소 | 입력값 변경 있으면 "변경사항이 사라져요" 다이얼로그 | - |

## 5. 저장 정책

- **저장 위치**: `profiles` 테이블
- **저장 시점**: "다음" 또는 "저장" 버튼 클릭 시 일괄 저장
- **저장 컬럼**: `nickname`, `birthdate`, `gender`, `job_field`, `career_level`, `main_concern`
- **완료 기록 (신규 진입 시)**: 저장 후 `profiles.profile_completed = true`로 UPDATE
- **수정 모드 시**: 변경된 필드만 UPDATE (`profile_completed`는 그대로 유지, 생년월일은 UPDATE 대상에서 제외)
- **임시 저장(이탈 대비)**: sessionStorage.draft_basic_info에 1분마다 백업, 다음 단계 진입 또는 저장 완료 시 삭제
- **입력값 → AI 시스템 프롬프트 컨텍스트로 주입**: 강점 인터뷰(05), 커리어 인터뷰(08) 시 활용. 수정 모드에서 변경된 직군/고민은 다음 AI 인터뷰(05/08/13) 시스템 프롬프트에 즉시 반영됨.

> ⚠️ **schema 검토 필요**: `profiles.birthdate`는 nullable로 설정되어 있으나, 본 화면에서는 필수 입력.  
> 닉네임 중복 검사 기능 제공 시 `profiles.nickname`에 UNIQUE 제약 추가 또는 서버 측 중복 체크 로직 필요 (현재 schema에 UNIQUE 제약 없음).

## 8. 접근성

- 모든 input에 <label>
- 필수/선택 표시 (별표 + "필수" 텍스트)
- 에러 메시지 aria-describedby로 input과 연결
- date picker 키보드 입력 가능
- dropdown은 native <select> 또는 ARIA combobox 패턴
- 수정 모드에서 잠금된 생년월일 필드는 aria-readonly 속성 + 시각적 비활성 표시
