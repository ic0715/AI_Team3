## 📋 Clarification Questions

### 1. 파일 분석 범위 및 방식

**Q1-1. 지원 파일 형식**
- 현재 프로토타입에는 "PDF · JPG · DOCX" 언급되어 있습니다
- 추가로 지원해야 할 형식이 있나요? (예: PNG, HWPX, TXT, JSON 등)
- 우선순위가 가장 높은 형식은 무엇인가요?

**Q1-2. 비정형 이력서 처리**
- "통일된 양식이 아니어도" 분석 가능하게 하려면, 어느 정도 수준의 정보 추출을 기대하시나요?
  - **Level 1**: 기본 정보만 (이름, 연락처, 현재 직무)
  - **Level 2**: 경력 타임라인 + 프로젝트 경험
  - **Level 3**: 기술 스택, 성과 지표, 소프트 스킬까지
- 실패 케이스 처리: 정보 추출이 불완전할 때 어떻게 하나요?
  - 사용자에게 수동 입력 요청?
  - 부분적으로 추출된 정보만 사용?

**Q1-3. 진단 결과 파일 (갤럽, MBTI 등)**
- 이런 파일들은 보통 이미지(스크린샷) 형태일 텐데:
  - OCR로 텍스트 추출 후 파싱해야 할까요?
  - 아니면 사용자가 직접 입력하는 필드를 더 강화할까요?
- 각 진단 도구마다 다른 포맷인데, 어떤 것들을 우선 지원하시나요?

### 2. Interview 연동 방식

**Q2-1. 동적 질문 생성**
업로드된 정보를 기반으로 질문을 커스터마이징하려면:
- **시나리오 A**: 미리 정의된 10개 질문 중 일부를 건너뛰기
  - 예: 이력서에서 "리더십 경험 3건" 발견 → "리더십 경험" 질문 생략
- **시나리오 B**: 질문 내용 자체를 맞춤화
  - 예: "OO 프로젝트에서 어떤 강점을 발휘했나요?" (구체적 프로젝트명 삽입)
- **시나리오 C**: 새로운 질문 추가
  - 예: 경력 공백 발견 시 → "2021-2022년에는 어떤 활동을 하셨나요?" 추가

어떤 방식을 원하시나요? 또는 혼합?

**Q2-2. 연동 타이밍**
- **옵션 A**: 파일 업로드 직후 즉시 분석 → "분석 중..." 로딩 → 커스터마이징된 질문 세트 표시
- **옵션 B**: S2 완료 후 S3 진입 시점에 백그라운드로 분석 → 질문 진행 중 동적으로 반영
- **옵션 C**: 모든 온보딩 완료 후 결과 생성 시점에만 활용

**Q2-3. 분석 결과 피드백**
파일 분석 후 사용자에게 어떤 피드백을 줘야 할까요?
```
예시:
"✓ 이력서에서 5개 프로젝트 경험을 발견했어요"
"✓ MBTI 결과(INTJ)를 확인했어요"
"⚠ 강점 진단 결과를 읽지 못했어요. 직접 입력해주시겠어요?"
```

### 3. UI/UX 디테일

**Q3-1. 파일 업로드 후 미리보기**
- 업로드된 파일의 미리보기(썸네일)를 보여줄까요?
- 추출된 정보를 바로 표시해서 "이렇게 이해했어요" 확인 단계가 필요한가요?

**Q3-2. 여러 파일 처리**
- 사용자가 이력서 1개 + 진단 결과 3개 등 여러 파일을 올릴 수 있습니다
- 각 파일의 역할을 자동 분류할까요? (이력서 vs 진단 결과 vs 기타)
- 아니면 사용자가 "이 파일은 이력서입니다" 선택하게 할까요?

**Q3-3. 건너뛰기 vs 필수**
현재는 "없으면 건너뛰어도 됩니다" 라고 되어 있는데:
- 파일 없이 진행 시 인터뷰 질문이 달라져야 할까요?
- 최소한의 정보 입력을 강제해야 할까요? (예: 직무, 경력년수만이라도)

### 4. 기술적 구현 범위

**Q4-1. AI 분석 백엔드**
- 실제로 Claude API를 호출해서 파일 분석을 하는 프로토타입을 만들까요?
- 아니면 프론트엔드 프로토타입 레벨에서 "분석 시뮬레이션"만 하면 될까요?

**Q4-2. 프라이버시/보안**
- 업로드된 파일을 서버에 저장할까요, 아니면 브라우저 메모리에서만 처리?
- 파일 크기 제한은? (예: PDF 5MB 이하)
