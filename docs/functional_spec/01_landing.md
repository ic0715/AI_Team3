# 01. 랜딩 / 코칭 합의

> 서비스를 처음 방문한 사용자에게 코칭 원칙(고객 주도·강점 기반·코칭 범위)을 안내하고, 동의 후 회원가입/로그인으로 자연스럽게 유도하는 화면.

## 1. 화면 개요

| 항목 | 내용 |
| --- | --- |
| 화면 ID | 01_landing |
| 페이즈 | ONBOARDING |
| 역할 | 서비스 진입, 코칭 방식 안내, 가입/로그인 유도 |
| 진입 경로 | 직접 URL 접속, 마케팅 링크, 검색 결과 |
| 다음 화면 | 02 로그인/회원가입 |

## 2. 진입 조건과 라우팅

사용자 상태별로 본 화면 노출 여부가 결정됨. 이미 로그인된 사용자는 본인의 상태에 맞는 화면으로 자동 리다이렉트.

| 사용자 상태 | 동작 |
| --- | --- |
| GUEST (비로그인) | 본 화면 정상 노출 |
| UNVERIFIED (이메일 미인증) | NEW01 이메일 인증으로 자동 리다이렉트 |
| ONBOARDING (온보딩 진행 중) | 마지막 진행 단계 화면으로 자동 리다이렉트 (아래 2.1 참조) |
| PAUSED (목표 일시중단 중) | 11 홈으로 자동 리다이렉트 (중단됨 배지 표시) |
| ACTIVE (12주 진행 중) | 11 홈으로 자동 리다이렉트 |
| COMPLETED (12주 완료) | NEW03 12주 완료 화면으로 자동 리다이렉트 |

> **상태 판별 우선순위:** ACTIVE → PAUSED → COMPLETED → ONBOARDING → UNVERIFIED 순으로 체크.  
> active 목표가 있으면 ACTIVE로 처리하여 11 홈으로 이동.

### 2.1 ONBOARDING 리다이렉트 세부 로직

ONBOARDING 상태인 경우, 아래 순서로 DB를 체크해 마지막으로 완료된 단계 다음 화면으로 이동.

```
1. profiles.profile_completed = false  →  03 기본 정보 입력
2. strength_analyses (is_latest=true) 없음  →  04 강점 진단 선택
3. career_interview_results 없음  →  07 커리어 인터뷰 안내
4. career_interview_results.recommended_goal_categories is null  →  09 커리어 인터뷰 결과
5. goals (active) 없음  →  10 목표 선택 (액션아이템 화면)
```

> `profiles.profile_completed` 컬럼: `profiles` 테이블에 `boolean DEFAULT false` 로 추가 필요.  
> 03 기본 정보 입력 완료 시 `true`로 UPDATE.  
> → **schema 반영 필요 항목** (spec-schema.md에 미기재)

### 2.2 COMPLETED 판별 기준

`goals.status = 'completed'` AND `goals.current_week >= goals.total_weeks`(기본값 12) 인 목표가 있고,  
현재 `active` 목표가 없는 경우 COMPLETED로 판별.

> active 목표가 새로 생성되면 ACTIVE 우선으로 처리됨.

## 3. UI 구성

### 3.1 헤더

- 좌측: CareerPT 로고 (SVG 웨이브 아이콘 + 워드마크)
- 로고 하단 서브타이틀: "나를 찾아가는 12주 AI 코칭 여정 ✨"
- 우측 로그인 링크: **없음** — 기존 사용자 진입은 하단 Footer 보조 버튼("이미 계정이 있어요")으로 통합

### 3.2 Hero 영역

- 눈썹(Eyebrow) 태그: "📋 코칭 합의" (본 화면이 코칭 합의 맥락임을 명시)
- 메인 타이틀: "이 대화는 답을 주기보다, 스스로 답을 찾도록 돕습니다."
- 설명 문구: "코칭을 시작하기 전에 오늘의 대화 방식과 범위를 짧게 합의해요. 주제, 행동, 성공 기준은 모두 **당신이 직접 선택**합니다."
- 시각: 별도 Hero 이미지 없음. 텍스트 중심 레이아웃. 애니메이션은 prefers-reduced-motion 대응

### 3.3 코칭 원칙 카드 (3장)

서비스의 코칭 철학을 3개 카드로 안내. MCC 코칭 원칙 기반.

| 카드 | 아이콘 | 본문 |
| --- | --- | --- |
| 고객 주도 | 🎯 | "AI 코치는 먼저 결론을 정하지 않아요. 당신이 말한 표현과 선택을 바탕으로 질문하고 반영합니다." |
| 강점 기반 | ⭐ | "갤럽 클리프턴 스트렝스 34개 테마는 확정 진단이 아니라, 대화에서 관찰된 강점 후보와 행동 설계 키워드로 활용해요." |
| 코칭 범위 | 🧭 | "커리어, 성장, 역할, 목표, 실행을 다뤄요. 심리치료, 법률, 의료, 재무 조언이 필요한 주제는 별도 전문가가 더 적합합니다." |

### 3.4 12주 여정 안내 타임라인

세로 스텝 형태. 각 단계는 숫자 원형 배지 + 연결선 + 제목 + 부제목으로 구성.

| 단계 | 제목 | 부제목 |
| --- | --- | --- |
| 1 | 강점 발견 | AI 인터뷰로 나만의 강점 테마 언어화 |
| 2 | 커리어 방향 설정 | 강점 기반으로 12주 목표 방향 결정 |
| 3 | 매주 액션 + 회고 | 1~12주차 동안 작은 실행과 AI 코칭 반복 |
| 4 | 12주 완주 후 다음 사이클 | 성취 요약 확인 + 새로운 여정 시작 |

> 4단계 배지는 강조 배경색(accent) 대신 연한 배경(accent-light) + accent 글자색으로 처리 (미완료 상태 표현)

### 3.5 Bottom CTA (Footer 고정 영역)

- **Primary**: "동의하고 시작하기 →" — Filled 버튼, 전체 너비 → 02 로그인/회원가입 (회원가입 탭)
- **Secondary**: "이미 계정이 있어요" — Outline 버튼, 전체 너비, Primary 아래 10px 간격 → 02 로그인/회원가입 (로그인 탭)

> "동의하고 시작하기" 카피는 본 화면에서 약관 동의 효력이 발생하지 않음. 실제 동의는 02에서 받음. 카피는 진입 동기 부여용.  
> 헤더에는 별도 "로그인" 링크가 없으며, 기존 사용자 진입은 Footer Secondary 버튼으로 일원화.

## 4. 기능 및 인터랙션

| 기능 | 동작 | 비고 |
| --- | --- | --- |
| Primary CTA 클릭 | 02 로그인/회원가입(회원가입 탭)으로 이동 | utm 파라미터 보존 |
| Secondary CTA 클릭 | 02 로그인/회원가입(로그인 탭)으로 이동 | 로그인 탭 활성 상태로 직행 |
| 스크롤 | 본문 스크롤, 헤더는 정적(스티키 아님) | Footer CTA는 페이지 하단 위치 |
| 카드 hover/tap | 미세한 그림자 변화 | 정보성 카드, 클릭 동작 없음 |

## 6. 데이터 정책

- **저장 데이터**: 없음
- **동의 효력**: 본 화면에서 발생하지 않음 (02에서 발생)
- **클라이언트 저장소 사용**: 없음

## 7. 분석 이벤트

| 이벤트 | 트리거 | 속성 |
| --- | --- | --- |
| landing_view | 페이지 로드 | utm_source, utm_medium, utm_campaign |
| landing_cta_clicked | Primary CTA("동의하고 시작하기") 클릭 | - |
| landing_login_clicked | Secondary CTA("이미 계정이 있어요") 클릭 | - |
| landing_scroll_depth | 스크롤 25/50/75/100% | depth |

## 8. 접근성 (WCAG 2.1 AA)

- 모든 카드/버튼에 키보드 포커스 가능, 포커스 인디케이터 명확
- Hero 일러스트에 alt 또는 aria-hidden="true" (장식용일 경우)
- 색약 사용자 대응: 카드 색상 외 아이콘으로 구분
- 글자 크기 200% 확대 시 레이아웃 깨지지 않음
- 명도 대비: 본문 4.5:1 이상, 대제목 3:1 이상

## 9. 성능 목표

- LCP 2.0초 이하 (Hero 이미지 우선 로딩)
- 초기 JS 번들 50KB(gzip) 이하
- 이미지: WebP/AVIF 포맷 + lazy load
- CLS 0.1 이하
