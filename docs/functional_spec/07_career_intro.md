# 07. 커리어 인터뷰 인트로

> 강점 결과를 확인한 유저가 커리어 방향 탐색으로 자연스럽게 전환하도록 동기를 부여하는 안내 화면.

## 1. 화면 개요

| 항목 | 내용 |
| --- | --- |
| 화면 ID | 07_career_intro |
| 페이즈 | DIRECTION |
| 역할 | 커리어 인터뷰 사전 안내, 강점 → 커리어 전환 동기 부여 |
| 이전 화면 | 06 강점 결과 (또는 15에서 재진입) |
| 다음 화면 | 08 커리어 인터뷰 |

## 2. 진입 조건

- 06에서 "다음" 클릭
- 또는 15에서 "커리어 방향 재설정" 요청 시 (이때 안내 문구 일부 변경)

## 3. UI 구성

### 3.1 상단 바

- 뒤로가기 → 06 또는 15
- 페이지 타이틀: "커리어 인터뷰"

### 3.2 Hero

- 이모지: 🗺
- 타이틀: "이제 커리어 방향을 찾아볼 차례예요"
- 서브: "강점을 어디에 어떻게 쓸지 함께 정리해요"
- 소요 시간 배지: "⏱ 약 8~12분 소요"

### 3.3 강점 요약 배지

- 06에서 도출된 Top 5 강점 chip 표시
- "이 강점들을 컨텍스트로 인터뷰가 진행돼요" 안내

### 3.4 인터뷰 안내 카드 (4개)

| 카드 | 내용 |
| --- | --- |
| 💼 현재 상황 | 직장 생활의 만족/불만족 요인 확인 |
| 🎯 미래 방향 | 5년 후 모습, 목표 |
| 🔥 동기·가치관 | 어떤 일에 에너지를 얻나요 |
| 📊 방향 추천 | 강점 + 답변 종합으로 5가지 방향 제안 |

### 3.5 안내 메시지

- "강점 분석 결과(Top 5)를 참고하여 질문이 구성돼요"
- "정답이 없는 질문이 많아요. 솔직하게 답해주세요"

### 3.6 Bottom CTA

- "인터뷰 시작하기 →"

## 4. 기능

- 시작 클릭 → 08 진입 (별도 세션 생성 없음 — 대화는 브라우저 메모리에서만 진행)
- sessionStorage에 진행 중인 대화가 있으면 이어하기 다이얼로그 (브라우저 내에서만 가능)
- `strength_analyses` (is_latest=true) 읽어서 인터뷰 prompt에 자동 포함

## 5. 데이터

- 읽기: `strength_analyses` (is_latest=true, Top 5 표시 + AI 프롬프트 컨텍스트용)
- 쓰기: 없음 (Static 전환 화면)

> ⚠️ **schema 불일치 수정**:
> - `strength_results` → `strength_analyses` (테이블명)
> - `coaching_sessions` 신규 생성 로직 제거 (v0.2에서 삭제된 테이블, 대화 원문 미저장 정책)

## 6. 예외 처리

| 상황 | 처리 |
| --- | --- |
| 강점 결과 미존재 | 04로 리다이렉트 + 안내 |
| 진행 중 세션 존재 | 이어하기 다이얼로그 |
| 세션 생성 실패 | 토스트 + 재시도 |
