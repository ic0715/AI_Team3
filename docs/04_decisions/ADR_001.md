# ADR 001 — 서비스 핵심 방향 결정

**날짜:** 2026-05-02  
**상태:** 확정 (v2 — 온보딩 구조 수정)  
**작성:** redesign/eunsang branch

---

## 배경

기존 프로토타입을 재점검하는 과정에서 서비스의 핵심 방향에 대한 근본적인 질문들이 제기됐다.

- 자기계발/목표달성 카테고리의 유사 서비스가 많은 상황에서 차별화 포인트가 불명확했다
- 팀의 자산(갤럽 코칭 자격증 2명, HR 출신 2명)이 서비스 설계에 충분히 반영되지 않았다
- 기존 온보딩의 AI 진단 리포트 구조가 ICF MCC 코칭 원칙과 충돌하고 있었다

---

## 결정 1: 서비스 핵심 정의 확정

> "나를 잘 모르거나, 알아도 막연한 직장인이  
> AI 코치와의 대화를 통해 자기 자신을 발견하고,  
> 그 발견이 커리어 행동으로 이어지게 하는 서비스"

**타겟 사용자:** 뭘 해야 할지 자체가 모호한 사람 (방향이 없어서 시작을 못 하는 사람)

**우리가 하는 것**
- 질문을 통한 자기발견
- 갤럽 강점 언어로 반영
- 발견이 커리어 행동으로 이어지는 연결

**우리가 하지 않는 것**
- AI가 먼저 분석 결과 제시
- 목표 로드맵 부여
- 컨설팅식 해결책 제안

---

## 결정 2: 온보딩 구조 확정

### 전체 흐름

```
회원가입
    ↓
기본설문 (최소 3개 항목)
    ↓
파일 업로드 화면
    ├── [파일 있음] 업로드 → 세션 2 바로 시작
    └── [파일 없음] "간단한 인터뷰로 대체할게요" →
            보완 인터뷰 (5개 질문) → 세션 1 (발견) → 세션 2 시작
```

### 기본설문 범위

코칭에 꼭 필요한 최소한의 맥락만 수집한다.

| 항목 | 내용 |
|---|---|
| 이름 | 닉네임 또는 이름 |
| 현재 직무/역할 | 한 줄 자유 입력 |
| 지금 가장 마음에 걸리는 것 | 한 줄 자유 입력 (첫 코칭 세션 오프닝과 연결) |

**폐기된 항목:** 나이, 회사명, 연차, 감정 선택지 등

### 파일 업로드

있으면 코칭 대화를 더 풍부하게 해주는 선택 사항이다.

- 갤럽 CliftonStrengths 결과지
- 이력서
- MBTI, 버크만 등 기타 진단 결과

파일이 없는 사용자는 보완 인터뷰로 동일한 정보를 수집한다.

### 보완 인터뷰 (파일 없는 사용자만)

갤럽 강점 리포트를 대체하는 5개 질문이다.
채팅 형식으로 하나씩 순서대로 진행한다.

| 순서 | 질문 | 탐색 목적 |
|---|---|---|
| Q1 | 일하면서 시간 가는 줄 모르고 몰입했던 순간이 있었나요? 어떤 상황이었는지 떠오르는 대로 말씀해주세요. | 에너지 원천 / 강점 도메인 파악 |
| Q2 | 주변 사람들이 나에게 자주 부탁하거나 "이건 네가 잘하잖아"라고 말하는 게 있나요? | 타인이 인식하는 강점 파악 |
| Q3 | 일할 때 절대 타협하기 싫은 것, 혹은 이것만큼은 지키고 싶다는 게 있나요? | 핵심 가치 / 신념 파악 |
| Q4 | 팀이나 사람들 사이에서 내가 자연스럽게 맡게 되는 역할이 있나요? (예: 분위기를 만드는 사람, 정리해주는 사람, 앞장서는 사람, 깊이 들어주는 사람) | 관계 / 영향력 스타일 파악 |
| Q5 | 지금까지 커리어에서 가장 뿌듯했던 순간을 하나 꼽는다면요? 어떤 상황이었고, 왜 그게 기억에 남으세요? | 성취 경험 탐색 (기본설문 Q3와 중복 방지) |

인터뷰 완료 후 AI가 내부적으로 갤럽 키워드 후보 3~5개를 잠정 설정하고,
이를 첫 코칭 세션의 배경 맥락으로 활용한다.

## 결정 3: 세션 구조 확정

### 세션 진입 경로
- 파일 업로드 유저: 온보딩 완료 → 세션 2 바로 시작
- 파일 없는 유저: 온보딩 → 보완 인터뷰 → 세션 1 → 세션 2

### 세션별 목표와 결과물

| 세션 | 목표 | 결과물 |
|---|---|---|
| 세션 1 (파일 없는 유저) | 나는 어떤 사람인가 | 키워드 1~3개 + 첫 번째 행동 |
| 세션 2 | 키워드가 커리어에서 어떤 의미인가 | 이번 주 액션아이템 1개 |
| 세션 3+ | 지난 행동 점검 + 다음은 | 다음 액션아이템 1개 |

매 세션마다 손에 잡히는 결과물이 나온다.
세션 간격은 앱이 강제하지 않는다. 사용자가 행동 후 스스로 시작한다.

### 오프닝 원칙 (기본설문 Q3 이어받기)
```
"[기본설문 Q3 답변]에 대해 좀 더 이야기해주실 수 있어요?
 그 상황이 지금 어떻게 느껴지세요?"
```
기본설문에서 이미 받은 답변을 AI가 기억하고 이어받는다.
같은 질문을 반복하지 않는다.

---

## 결정 4: 갤럽 키워드는 출력이 아닌 반영

AI가 먼저 갤럽 키워드를 제시하지 않는다.
대화를 통해 사용자가 발견한 것을 AI가 언어로 반영해준다.

**이전 방식 (폐기)**
```
AI 분석 → "당신의 강점은 전략, 분석, 배움입니다" → "어떻게 느껴지세요?"
```

**새로운 방식**
```
대화 → 패턴 포착 → 반영 → 사용자 공명 확인 → 키워드 연결
```

---

## 결정 5: AI 진단 리포트 폐기

역량 갭 분석, 1/3/5년 목표 로드맵을 첫 화면에서 없앤다.

**이유**
- ICF MCC 원칙 위반: AI가 먼저 결론을 내리는 구조
- 사용자가 아직 발견하지도 않은 것을 AI가 먼저 정리해서 주는 구조

**대체**
- 세션 1에서 사용자가 직접 키워드를 선택하고 확정
- 세션 2부터 액션아이템 도출
- 세션 3+에서 지난 액션 점검 + 다음 액션아이템

---

## 이 결정들의 차별화 포인트

| 다른 앱 | 이 서비스 |
|---|---|
| 설문 → 분석 → 결과 제시 | 대화 → 발견 → 사용자가 직접 정리 |
| 사용자를 피험자로 | 사용자를 탐험가로 |
| AI가 답을 줌 | AI가 질문으로 공간을 만들어 줌 |
| MBTI식 라벨링 | 코칭식 자기발견 |

이 설계를 구현할 수 있는 건 팀이 가진 MCC + 갤럽 코칭 전문성 덕분이다.
