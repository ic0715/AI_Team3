# 05. 강점 인터뷰 (AI 채팅)

> AI 코치와 멀티턴 대화를 통해 유저의 핵심 강점 테마를 탐색·언어화. 코어 6개 + 동적 follow-up 구조. 갤럽 CliftonStrengths 34 테마 매핑 기반.

## 3. UI 구성

### 3.1 상단 바

- 뒤로가기 (이탈 시 진행 상태 자동 저장 안내)
- 페이지 타이틀: "강점 인터뷰"
- 진행률: "Q3 / 6" + Progress Bar (코어 질문 기준)

### 3.2 채팅 영역

#### AI 메시지

- 코치 아바타 + 이름 ("CareerPT 코치")
- 질문 텍스트 + 메시지 시간

#### 사용자 메시지

- 우측 정렬 버블 + 메시지 시간

#### Typing Indicator

- AI 응답 대기 중 점 3개 애니메이션

### 3.3 입력 영역

- textarea (자동 높이, 최대 120px)
- Enter 전송, Shift+Enter 줄바꿈
- 전송 버튼
- "인터뷰 완료하기" CTA (코어 6개 모두 답변 시 활성화)
- 사용자 입력 중 PII 패턴(주민번호, 카드번호) 감지 시 경고

### 3.4 상단 도움말

- "그만하기" 버튼 (이탈)
- "?" 도움말 → 인터뷰 팁 모달

## 4. 인터뷰 코어 질문 (6개)

| Q | 질문 | 목적 |
| --- | --- | --- |
| Q1 | 최근 일이나 활동에서 시간 가는 줄 모르고 몰입했던 순간을 알려주세요 | 흥미·강점 신호 수집 |
| Q2 | 동료/지인이 당신에게 자주 부탁하거나 칭찬하는 것은? | 외부 관점 강점 |
| Q3 | 새로운 일을 시작할 때, 당신이 가장 먼저 하는 행동은? | 행동 패턴 |
| Q4 | 어떤 환경/상황에서 가장 에너지가 올라가나요? | 동기 패턴 |
| Q5 | 최근에 뿌듯했던 성취 1가지를 알려주세요 | 자기 인식 강점 |
| Q6 | 반대로, 잘 안되거나 피하고 싶은 일은? | 약점·비강점 영역 |

### Follow-up 정책

AI가 답변을 평가하여 다음 조건 중 하나에 해당하면 follow-up 질문 자동 생성:

| 트리거 | 예시 답변 | Follow-up 예시 |
| --- | --- | --- |
| 답변 길이 30자 미만 | "잘 모르겠어요" | "구체적인 상황 하나를 떠올려본다면 어떤가요?" |
| 추상적 표현만 사용 | "좋은 환경이요" | "최근에 그런 환경을 경험했던 순간이 있다면?" |
| 모순 탐지 | Q1과 Q3 답변 충돌 | "앞에서는 X라고 하셨는데, 지금 답변은 Y네요. 어떻게 연결되나요?" |
| 핵심 키워드 부재 | 강점 인터뷰인데 행동 묘사 없음 | "그때 어떤 행동을 했는지 자세히 알려주실 수 있을까요?" |

- Follow-up 제한: 코어 질문 1개당 최대 2회. 그 이상은 강제 다음 단계 진행. 인터뷰당 총 follow-up 최대 12개.

## 5. AI 시스템 프롬프트 설계

- **코칭 기반**: MCC(Master Certified Coach) 원칙 - 코치가 먼저 결론 제시 금지, 유저 스스로 답을 찾도록 질문 중심 진행
- **강점 프레임**: 갤럽 CliftonStrengths 34 테마를 참조, "확정 진단"이 아닌 "관찰된 강점 후보"로 활용
- **개인화**: 03 기본정보(직군·나이·고민)와 이전 대화 맥락을 시스템 프롬프트에 포함
- **출력 형식**: 인터뷰 종료 시 AI가 JSON 구조로 결과 반환 → 파싱 후 Supabase 저장
- **컨텍스트 누적**: 멀티턴 대화 - 대화 히스토리 전체 누적 전달

## 6. 기능 및 인터랙션

| 기능 | 동작 |
| --- | --- |
| 메시지 전송 | sessionStorage 백업 → 서버 저장 → AI 응답 요청 |
| AI 응답 | 스트리밍 응답 (가능 시) |
| Follow-up 트리거 | AI가 답변 평가 후 자동 |
| 진행률 갱신 | 코어 질문 완료 시 +1 |
| 자동 스크롤 | 새 메시지 생성 시 하단으로 |
| 그만하기 | "지금까지 답변은 저장돼요" 안내 후 15 또는 직전 화면 |
| 5분 무응답 | "잠시 자리 비우셨나요?" 다이얼로그 |
| 30분 무응답 | 자동 저장 후 세션 paused |
| 인터뷰 완료 | AI가 강점 JSON 반환 → strength_results 저장 → 06으로 자동 이동 |

## 7. 자동 저장 정책

- **매 메시지마다** interview_messages에 즉시 INSERT
- **사용자 입력 중** sessionStorage.draft_message_{session_id}에 백업
- **전송 실패 시** 재시도 큐(최대 3회, 지수 백오프 1s/3s/9s)
- **3회 실패 시** 사용자에게 안내 + 재시도 버튼
- **다른 디바이스 진입 시** 마지막 메시지 시점 기준 자동 동기화

## 8. 인터뷰 재개 (Resume)

사용자가 인터뷰 도중 이탈 후 다시 진입한 경우:

- **1단계**: interview_sessions WHERE user_id=? AND type=strength AND status=in_progress 검색
- **2단계**: 세션 존재 시 다이얼로그 표시
  -    · "이어서 하기" → 기존 세션 메시지 복원, 마지막 AI 질문 표시
  -    · "처음부터" → 기존 세션 status=abandoned 처리, 새 세션 시작
- **3단계**: 세션 부재 시 새 세션 생성

## 9. 예외 처리

| 상황 | 처리 |
| --- | --- |
| 빈 메시지 전송 시도 | 차단 |
| AI API 응답 지연(>5초) | 로딩 인디케이터 + 타임아웃 안내 |
| AI API 오류 (5xx) | 재시도 버튼, 3회 연속 실패 시 고객 문의 안내 |
| AI 응답 JSON 파싱 불가 | 오류 로그 기록, "잠시 후 다시 시도" 안내 |
| 네트워크 단절 중 입력 | 큐 적재, 연결 시 자동 전송 |
| 위기 신호 감지 (자살/자해 등) | 즉시 인터뷰 일시 중단 + 자살예방상담전화 1393 안내 + 고객센터 |
| PII 입력 감지 | 입력란 위 경고 + 마스킹 안내 |
| 부적절한 AI 응답 (필터 통과 실패) | 안전 메시지로 대체 |
| Follow-up 무한 루프 | 코어 질문당 최대 2회 후 강제 진행 |
| 세션 만료 (JWT) | 토큰 갱신 후 재시도 |
| 동시 다중 디바이스 | 마지막 메시지 시점 기준 동기화 |
