# 12. 회고 (평일/주말)

> 평일에는 일일 메모를 작성하고, 주말에는 한 주 메모를 기반으로 주간 회고를 완성.

## 1. 화면 개요

| 항목 | 내용 |
| --- | --- |
| 화면 ID | 12_reflect |
| 페이즈 | MAINTAIN |
| 역할 | 평일 메모 + 주말 회고 |
| 진입 경로 | 탭바 "회고", 11 알림 카드 |
| 다음 화면 | 13 회고 코칭 (주말 회고 후) |

## 2. 진입 조건

- 사용자 상태 = ACTIVE
- 탭바 "회고" 선택
- 자동 분기: 토/일 = 주말 모드, 그 외 = 평일 모드 (수동 토글 가능)

## 3. UI 구성

### 3.1 공통 상단 바

- 페이지 타이틀: "회고"
- 우측: 평일/주말 모드 수동 토글

### 3.2 평일 모드

#### 헤더

- "오늘은 어땠어요?"
- 오늘 요일 표시

#### 이번 주 액션 카드

- 현재 주차 액션 텍스트 + 진행률

#### 메모 입력

- textarea (최대 500자, 글자수 표시)
- "저장" 버튼

#### 메모 리스트 (이번 주, 월~금)

- 요일 + 날짜 + 내용
- 수정/삭제 버튼 (3일 이내만 수정 가능, 그 이후는 읽기 전용)

#### 회고 코칭 카드 (주중 약하게 노출)

- "이번 주 회고를 미리 시작해볼까요?" → 13

### 3.3 주말 모드

#### 헤더

- "한 주를 마감해요"
- 주차 + 요일

#### 이번 주 액션 요약 카드

- 목표 vs 수행 비교 (예: "3/7 일 완료")
- 간단 시각화 (도넛 또는 막대)

#### 평일 메모 컨텍스트

- 주중 메모 리스트 자동 노출

#### 회고 입력

- 한 주 회고 textarea (최대 1000자)
- 가이드 prompt (선택): "이번 주 어떤 점이 좋았나요?"

#### 저장 + CTA

- "저장만 하기"
- Primary "코치와 이야기 나누기 →" → 13

## 4. 기능

| 기능 | 동작 |
| --- | --- |
| 모드 자동 분기 | 토/일 = 주말, 그 외 = 평일 (사용자 토글로 변경 가능) |
| 메모 저장 | weekly_memos 즉시 저장 (week_num, day_of_week, memo_text, memo_date) |
| 메모 수정 | 3일 이내만, 그 외 읽기 전용 |
| 메모 삭제 | 확인 다이얼로그 |
| 주간 회고 저장 | weekly_memos에 day_of_week=weekend로 저장 |
| 코치와 이야기 나누기 | 13 진입 |

## 5. 예외 처리

| 상황 | 처리 |
| --- | --- |
| 메모 미작성 상태로 주간 회고 시도 | "최소 1개 메모를 작성해주세요" 안내 |
| 메모 미입력 저장 | 차단, 포커스 유지 |
| 메모 500자 초과 | 입력 제한 |
| 주간 회고 저장 실패 | 로컬 임시 저장 후 재전송 유도 |
| 회고 1000자 초과 | 입력 제한 |
| 모드 강제 변경 | 토글로 가능 (예: 토요일에 평일 메모) |
| HTML 입력 | XSS 방지 위해 escape 후 저장 |

## 6. 분석 이벤트

| memo_edited | - |

| weekly_reflection_saved | week_num, length |
| coaching_initiated_from_reflect | - |

## 7. 접근성

- textarea는 적절한 레이블링
- 저장 성공/실패는 aria-live로 알림
- 모드 토글은 토글 버튼 ARIA 패턴
