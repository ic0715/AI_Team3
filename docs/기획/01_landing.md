# 01. 랜딩 / 코칭 합의

> 서비스를 처음 방문한 사용자에게 코칭 원칙(고객 주도·강점 기반·코칭 범위)을 안내하고, 동의 후 회원가입/로그인으로 자연스럽게 유도하는 화면.

## 1. 화면 개요

| 항목 | 내용 |
| --- | --- |
| 화면 ID | 01_landing |
| 페이즈 | ONBOARDING |
| 역할 | 서비스 진입, 코칭 방식 안내, 가입/로그인 유도 |
| 진입 경로 | 직접 URL 접속, 마케팅 링크, 검색 결과 |
| 다음 화면 | 02 로그인/회원가입 |

## 2. 진입 조건과 라우팅

사용자 상태별로 본 화면 노출 여부가 결정됨. 이미 로그인된 사용자는 본인의 상태에 맞는 화면으로 자동 리다이렉트.

| 사용자 상태 | 동작 |
| --- | --- |
| GUEST (비로그인) | 본 화면 정상 노출 |
| UNVERIFIED (이메일 미인증) | NEW01 이메일 인증으로 자동 리다이렉트 |
| ONBOARDING (온보딩 진행 중) | 마지막 진행 단계 화면으로 자동 리다이렉트 |
| ACTIVE (12주 진행 중) | 11 홈으로 자동 리다이렉트 |
| COMPLETED (12주 완료) | NEW03 12주 완료 화면으로 자동 리다이렉트 |

## 3. UI 구성

### 3.1 헤더

- 좌측: CareerPT 로고 (브랜드 아이콘 + 워드마크)
- 우측: "로그인" 텍스트 링크 → 02 로그인/회원가입 (로그인 탭)

### 3.2 Hero 영역

- 메인 카피: "방향이 없는 당신을 위한 AI 커리어 코치"
- 서브 카피: "당신의 강점에서 시작해 12주의 작은 실행으로 변화를 만들어요"
- 시각: 일러스트 또는 그라데이션 배경. 애니메이션은 prefers-reduced-motion 대응

### 3.3 코칭 원칙 카드 (3장)

서비스의 코칭 철학을 3개 카드로 안내. MCC 코칭 원칙 기반.

| 카드 | 내용 |
| --- | --- |
| 고객 주도 | "AI는 답을 정해주지 않아요. 당신이 답을 찾도록 좋은 질문을 던져요" |
| 강점 기반 | "약점을 고치는 대신 강점을 키우는 방식으로 접근해요" |
| 범위와 한계 | "직업·진로·역량 코칭을 다루며, 의료·법률·심리치료는 다루지 않아요" |

### 3.4 12주 여정 안내 타임라인

- **1단계**: 강점 발견 (1주차 시작 전)
- **2단계**: 커리어 방향 설정 (1주차 시작 전)
- **3단계**: 매주 작은 액션 + 회고 (1~12주차)
- **4단계**: 12주 완주 후 다음 사이클 (12주차 이후)

### 3.5 Bottom CTA

- Primary: "동의하고 시작하기 →" → 02 로그인/회원가입 (회원가입 탭)
- Secondary: "이미 계정이 있어요" → 02 로그인/회원가입 (로그인 탭)

> "동의하고 시작하기" 카피는 본 화면에서 약관 동의 효력이 발생하지 않음. 실제 동의는 02에서 받음. 카피는 진입 동기 부여용.

## 4. 기능 및 인터랙션

| 기능 | 동작 | 비고 |
| --- | --- | --- |
| Primary CTA 클릭 | 02 로그인/회원가입(가입 탭)으로 이동 | utm 파라미터 보존 |
| 헤더 로그인 클릭 | 02 로그인/회원가입(로그인 탭)으로 이동 | - |
| 스크롤 | 본문 스크롤, 헤더는 sticky 또는 정적 | 디자인 협의 |
| 카드 hover/tap | 미세한 그림자 변화 | 정보성 카드, 클릭 동작 없음 |

## 5. 예외 처리

| 상황 | 처리 |
| --- | --- |
| CTA 클릭 후 라우팅 실패 | 재시도 토스트 노출 |
| 페이지 로딩 실패 | NEW06 일반 오류 화면 |
| JavaScript 비활성 | 최소 콘텐츠는 SSR 또는 noscript fallback |
| 이미 로그인된 사용자 접근 | 사용자 상태 기반 리다이렉트(2번 항목 참조) |

## 6. 데이터 정책

- **저장 데이터**: 없음
- **동의 효력**: 본 화면에서 발생하지 않음 (02에서 발생)
- **클라이언트 저장소 사용**: 없음

## 7. 분석 이벤트

| 이벤트 | 트리거 | 속성 |
| --- | --- | --- |
| landing_view | 페이지 로드 | utm_source, utm_medium, utm_campaign |
| landing_cta_clicked | Primary CTA 클릭 | cta_position(top/bottom) |
| landing_login_clicked | 헤더 로그인 클릭 | - |
| landing_scroll_depth | 스크롤 25/50/75/100% | depth |

## 8. 접근성 (WCAG 2.1 AA)

- 모든 카드/버튼에 키보드 포커스 가능, 포커스 인디케이터 명확
- Hero 일러스트에 alt 또는 aria-hidden="true" (장식용일 경우)
- 색약 사용자 대응: 카드 색상 외 아이콘으로 구분
- 글자 크기 200% 확대 시 레이아웃 깨지지 않음
- 명도 대비: 본문 4.5:1 이상, 대제목 3:1 이상

## 9. 성능 목표

- LCP 2.0초 이하 (Hero 이미지 우선 로딩)
- 초기 JS 번들 50KB(gzip) 이하
- 이미지: WebP/AVIF 포맷 + lazy load
- CLS 0.1 이하
