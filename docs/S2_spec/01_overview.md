## 🎯 핵심 목표

### 문제 정의
- 사용자가 이력서, 진단 결과 등 **비정형 파일**을 업로드
- AI가 파일을 분석하여 **구조화된 데이터** 추출
- 추출된 정보를 기반으로 **Interview 질문을 동적으로 커스터마이징**

### 성공 지표
- ✅ 파일 분석 정확도: 기본 정보 **95%** 이상
- ✅ 분석 완료 시간: 파일당 **3-5초** 이내
- ✅ 사용자 만족도: "이렇게 이해했어요" 확인 후 **80%** 이상 "맞음" 선택
- ✅ Interview 커스터마이징: 평균 **2-3개 질문** 축소 또는 맞춤화

## ✅ 확정된 의사결정

### 1. 파일 분석 범위 (Q1)

| 항목 | 결정 사항 |
|------|----------|
| **지원 파일 형식** | PDF, DOCX, JPG |
| **비정형 이력서 처리** | Level 3 (기술 스택 + 성과 지표 + 소프트 스킬까지) |
| **실패 케이스** | 추출된 정보만 사용 (TBD - 향후 개선) |
| **진단 도구 우선순위** | Phase 1: MBTI, 갤럽 강점 |

### 2. Interview 연동 방식 (Q2)

| 항목 | 결정 사항 |
|------|----------|
| **동적 질문 생성** | **시나리오 A + B 혼합**<br>• A: 질문 건너뛰기 (정보 충분 시)<br>• B: 질문 내용 맞춤화 (구체적 정보 삽입) |
| **연동 타이밍** | **옵션 A (즉시 분석)** - 파일 업로드 직후 분석 → 결과 표시 |
| **분석 결과 피드백** | **2단계 피드백**<br>• Stage 1: 파일별 즉시 결과<br>• Stage 2: 교차 분석 종합 |

### 3. UI/UX 디테일 (Q3)

| 항목 | 결정 사항 |
|------|----------|
| **파일 업로드 후** | 썸네일 + 추출 데이터 표시 + 인라인 수정 기능 |
| **여러 파일 처리** | AI 자동 분류 + 사용자 확인 (맞음/변경) |
| **건너뛰기 정책** | 유연한 3-Tier 접근<br>• Tier 1: 완전 건너뛰기<br>• Tier 2: 부분 업로드 (권장)<br>• Tier 3: 완전 업로드 (최적) |

### 4. 기술 구현 (Q4)

| 항목 | 결정 사항 |
|------|----------|
| **AI 분석 백엔드** | **실제 Claude API 호출**<br>• Vision API (이미지)<br>• Documents API (PDF) |
| **보안/프라이버시** | **하이브리드 방식**<br>• 브라우저: Base64 변환<br>• 백엔드: 프록시 역할 (파일 미저장)<br>• DB: 추출 데이터만 저장 |
| **파일 크기 제한** | PDF 5MB, JPG 3MB, DOCX 5MB |

## 📊 추출 데이터 개요

### 이력서에서 추출
```
Level 1 (기본):
  - 이름, 연락처, 현재 직무/회사

Level 2 (경력):
  - 경력 타임라인 (회사, 직무, 기간)
  - 프로젝트 경험 (이름, 역할, 성과)

Level 3 (심화):
  - 기술 스택, 도메인 전문성
  - 정량적 성과 지표 (%, 숫자)
  - 소프트 스킬 추론 (리더십, 커뮤니케이션 등)
```

### 갤럽 강점 진단에서 추출
```
- Top 5 강점 (한글명 + 영문명)
- 카테고리 (E/I/R/T)
- 각 강점별 설명
```

### MBTI에서 추출
```
- 4글자 타입 (예: INTJ)
- 각 지표별 점수 (선택적)
```

## 🔄 핵심 플로우

```
1. 파일 업로드
   └─> 파일명/내용 기반 자동 분류
       └─> 사용자 확인 (맞음/변경)

2. 분석 시작
   └─> 로딩 3-5초
       └─> Stage 1: 파일별 결과 표시
           └─> Stage 2: 교차 분석 결과

3. Interview 커스터마이징
   └─> 시나리오 A: 질문 건너뛰기
       └─> 시나리오 B: 질문 내용 맞춤화
           └─> 새 질문 추가 (경력 공백 등)

4. S3 Interview 진입
   └─> 맞춤화된 질문 세트로 진행
```

## ⚠️ 알려진 제약사항

1. **OCR 정확도**: 이미지 기반 진단 결과는 PDF 대비 정확도 낮음 (약 70-80%)
2. **파일 크기**: 5MB 초과 파일은 업로드 불가
3. **언어**: 현재 한글만 지원 (영문은 Phase 2)
4. **동시 처리**: 한 번에 최대 5개 파일까지
