# 사용자 흐름 (User Flow)

**최종 업데이트:** 2026-05-02  
**상태:** 확정 (ADR_001 v2 기반)

---

## 전체 흐름

```
[1단계] 회원가입
    ↓
[2단계] 기본설문 (3개 항목)
    ↓
[3단계] 파일 업로드 화면
    ├── 파일 있음 → 업로드 → [세션 2] 바로 시작
    └── 파일 없음 → "간단한 인터뷰로 대체할게요" 선택
                        ↓
                   [3-B] 보완 인터뷰 (5개 질문)
                        ↓
                   [세션 1] 발견 세션
                   키워드 1~3개 + 첫 번째 행동
    ↓
[세션 2] 연결 세션
키워드-커리어 연결 + 이번 주 액션아이템
    ↓
[세션 3+] 행동 점검 세션
지난 액션 점검 + 다음 액션아이템
(사용자가 행동 후 스스로 시작)
```

### 세션별 결과물 요약

| 세션 | 목표 | 결과물 |
|---|---|---|
| 세션 1 (파일 없는 유저) | 나는 어떤 사람인가 | 키워드 1~3개 + 첫 번째 행동 |
| 세션 2 | 키워드가 커리어에서 어떤 의미인가 | 이번 주 액션아이템 1개 |
| 세션 3+ | 지난 행동 점검 + 다음은 | 다음 액션아이템 1개 |

매 세션마다 손에 잡히는 결과물이 나온다.
세션 간격은 앱이 강제하지 않는다. 사용자가 행동 후 스스로 시작한다.

---

## 1단계 — 회원가입

- 이메일 / 비밀번호
- 구글 소셜 로그인
- 이름 (닉네임)

---

## 2단계 — 기본설문

코칭에 꼭 필요한 최소한의 맥락만 수집한다.
3개 항목, 모두 자유 입력이다.

```
1. 어떻게 불러드릴까요?
   [이름 또는 닉네임 입력]

2. 현재 어떤 일을 하고 계세요?
   [한 줄 자유 입력 / 예: PM, 마케터, 개발자...]

3. 요즘 일이나 커리어에서 가장 마음에 걸리는 게 뭔가요?
   [한 줄 자유 입력]
```

3번 항목은 첫 코칭 세션의 오프닝 질문과 자연스럽게 연결된다.

**폐기된 항목 (기존 대비)**
- 나이
- 소속/회사
- 연차
- 감정 선택지
- 관심사/해결하고 싶은 문제 (별도 항목)

---

## 3단계 — 파일 업로드 화면

### 화면 구성

```
┌─────────────────────────────────────────┐
│  강점 리포트나 이력서가 있으면           │
│  업로드해주세요                          │
│                                         │
│  업로드 가능한 자료                      │
│  · 갤럽 CliftonStrengths 결과지         │
│  · 이력서                               │
│  · MBTI, 버크만 등 기타 진단 결과       │
│                                         │
│        [📄 파일 업로드]                 │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  따로 파일이 없어요.                    │
│  간단한 인터뷰로 대체할게요.  →         │
│                                         │
└─────────────────────────────────────────┘
```

### 파일이 있는 경우

업로드 완료 → 바로 4단계(첫 코칭 세션)로 이동

업로드된 파일은 코칭 대화의 배경 맥락으로만 활용한다.
AI가 먼저 분석 결과를 제시하지 않는다.

### 파일이 없는 경우

"간단한 인터뷰로 대체할게요" 선택 → 3-B 보완 인터뷰로 이동

---

## 3-B단계 — 보완 인터뷰 (파일 없는 사용자)

갤럽 강점 리포트를 대체하는 5개 질문이다.
채팅 형식으로 하나씩 진행한다.

### Q1 — 에너지 원천
```
일하면서 시간 가는 줄 모르고 몰입했던 순간이 있었나요?
어떤 상황이었는지 떠오르는 대로 말씀해주세요.
```
→ 탐색: 어떤 활동에서 에너지가 올라오는가 (배움/발상/성취/행동 등)

### Q2 — 타인이 인식하는 강점
```
주변 사람들이 나에게 자주 부탁하거나
"이건 네가 잘하잖아"라고 말하는 게 있나요?
```
→ 탐색: 자신이 잘 인식 못 하는 강점 (갤럽의 "기여하는 것"에 해당)

### Q3 — 핵심 가치
```
일할 때 절대 타협하기 싫은 것,
혹은 이것만큼은 지키고 싶다는 게 있나요?
```
→ 탐색: 신념/책임/공정성/연결성 등 가치 기반 키워드

### Q4 — 관계/역할 스타일
```
팀이나 사람들 사이에서 내가 자연스럽게 맡게 되는 역할이 있나요?
예를 들어, 분위기를 만드는 사람, 정리해주는 사람,
앞장서는 사람, 깊이 들어주는 사람 같은 것이요.
```
→ 탐색: 관계 구축 / 영향력 도메인 스타일

### Q5 — 성취 경험 탐색

```
지금까지 커리어에서 가장 뿌듯했던 순간을 하나 꼽는다면요?
어떤 상황이었고, 왜 그게 기억에 남으세요?
```
→ 성취 경험과 에너지 원천 탐색
→ Q1(몰입)은 과정, Q5(뿌듯함)는 결과 — 다른 각도
→ 기본설문 Q3("마음에 걸리는 것")와 중복 방지

인터뷰 완료 후 AI 내부적으로 갤럽 키워드 후보 3~5개 잠정 설정.
→ 4단계(첫 코칭 세션)로 이동

---

## 세션 1 — 발견 세션 (파일 없는 유저)

**목표:** 갤럽 키워드 1~3개 발견
**시간:** 3분 / 5턴

### 오프닝 (기본설문 Q3 이어받기)
```
"[기본설문 Q3 답변]에 대해 좀 더 이야기해주실 수 있어요?
 그 상황이 지금 어떻게 느껴지세요?"
```
기본설문에서 이미 받은 답변을 AI가 기억하고 이어받는다.
같은 질문을 반복하지 않는다.

### 5턴 구조
```
Turn 1 — 열기: 오늘 가장 중요한 것 하나 잡기
Turn 2 — 탐색: 감정과 맥락 파악
Turn 3 — 심화: 패턴 발견
Turn 4 — 반영: 사용자가 자신을 새롭게 보는 순간
Turn 5 — 연결: 키워드 제시 + 첫 번째 행동
```

### 세션 1 결과물
```
→ 사용자가 직접 선택한 키워드 1~3개
→ 사용자가 직접 정한 첫 번째 행동 1개
```

---

## 세션 2 — 연결 세션

**목표:** 키워드를 커리어 고민과 연결, 액션아이템 도출
**시간:** 3분 / 5턴
**진입:** 파일 업로드 유저는 온보딩 직후 / 파일 없는 유저는 세션 1 이후

### 5턴 구조
```
Turn 1 — 이전 연결: 지난 키워드 + 첫 행동 결과 확인
Turn 2 — 탐색: 키워드가 커리어에서 어떻게 작동하는지
Turn 3 — 장애물: 키워드 발휘를 막는 것 파악
Turn 4 — 가능성: 장애물을 우회하는 작은 가능성 발견
Turn 5 — 행동 설계: 이번 주 액션아이템 확정 (사용자가 직접 정함)
```

### 세션 2 결과물
```
→ 이번 주 액션아이템 1개 (언제, 어떤 상황에서 할지까지 구체화)
```

---

## 세션 3+ — 행동 점검 세션

**목표:** 지난 액션 점검 + 다음 액션아이템 도출
**시간:** 3분 / 5턴
**진입:** 사용자가 행동을 해보고 스스로 시작 (앱이 강제하지 않음)

### 5턴 구조
```
Turn 1 — 점검: 지난 액션 어떻게 됐는지 탐색 (평가 아님)
Turn 2 — 배움: 경험에서 배운 것 발견
Turn 3 — 키워드 연결: 배움을 키워드와 연결
Turn 4 — 다음 가능성: 다음 행동 방향 탐색
Turn 5 — 행동 확정: 다음 액션아이템 1개 (사용자가 직접 정함)
```

### 세션 3+ 결과물
```
→ 지난 액션에서 배운 것 1개
→ 다음 액션아이템 1개
```

## 메인 앱

### 홈
- 내 키워드 (누적)
- 현재 액션아이템
- 다음 세션 시작하기

### 체크인 알림
```
"오늘 일하면서 어떤 순간이 기억에 남으세요?"
```
→ 자연스럽게 다음 세션으로 유도

### 회고
- Keep / Problem / Try
- AI가 키워드와 연결되는 패턴 반영

## 이전 구조와 비교

### 이전 구조 (폐기)
```
회원가입 → 기본설문(상세) → 파일업로드
→ AI 인터뷰 10개 → AI 진단 리포트
(종합요약 / 강점개선 / 키워드 / 로드맵 / 역량갭)
→ 코칭 세션
```

### 새로운 구조
```
회원가입 → 기본설문(3개) → 파일업로드 or 보완인터뷰(5개)
→ 세션 1 (발견: 키워드 + 첫 행동)
→ 세션 2 (연결: 액션아이템)
→ 세션 3+ (점검: 다음 액션아이템)
```

핵심 차이
- AI가 결과를 주는 구조 → 사용자가 발견하는 구조
- 대화만 하고 끝나는 세션 → 매 세션마다 결과물이 나오는 구조
