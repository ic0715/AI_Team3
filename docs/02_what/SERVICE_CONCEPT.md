# 서비스 핵심 개념

**최종 업데이트:** 2026-05-02  
**상태:** 확정 (ADR_001 v2 기반)

---

## 한 줄 정의

> 나를 잘 모르거나, 알아도 막연한 직장인이  
> AI 코치와의 대화를 통해 자기 자신을 발견하고,  
> 그 발견이 커리어 행동으로 이어지게 하는 서비스

---

## 타겟 사용자

**핵심 타겟: C 유형**
뭘 해야 할지 자체가 모호한 사람.
방향이 없어서 시작을 못 하는 직장인.

**C 유형의 특징**
- 머리로는 뭔가 해야 한다는 걸 알지만 몸이 안 따라간다
- 무엇부터 시작해야 할지 막연하다
- 자신이 뭘 원하는지 자체가 불분명하다
- 혼자서는 정리가 안 된다

---

## 온보딩 구조

```
회원가입
    ↓
기본설문 (3개 항목만)
  · 이름
  · 현재 직무/역할
  · 지금 가장 마음에 걸리는 것
    ↓
파일 업로드 화면
    ├── [파일 있음] → 첫 코칭 세션 시작
    └── [파일 없음] → 보완 인터뷰(5개) → 첫 코칭 세션 시작
```

파일 업로드와 보완 인터뷰는 갤럽 강점 정보를 확보하기 위한 두 가지 경로다.
어느 경로든 첫 코칭 세션의 품질은 동일하다.

---

## 우리가 하는 것

**1. 최소한의 정보 수집으로 빠른 시작**
긴 설문 없이 3개 항목만으로 온보딩을 시작한다.
강점 정보는 파일 업로드 또는 보완 인터뷰로 확보한다.

**2. 질문을 통한 자기발견**
AI가 MCC 수준의 질문을 통해 사용자가 스스로 자신을 발견하게 돕는다.
답을 주지 않는다. 공간을 만들어준다.

**3. 갤럽 언어로 반영**
대화 중 드러나는 패턴을 갤럽 CliftonStrengths 34개 키워드로 연결한다.
AI가 먼저 분석해서 주는 게 아니라, 사용자가 발견한 것에 언어를 입혀준다.

**4. 커리어 행동으로 연결**
발견이 오늘의 첫 걸음으로 이어지게 돕는다.
거창한 로드맵이 아니라, 지금 당장 할 수 있는 구체적인 한 가지.

---

## 우리가 하지 않는 것

- AI가 먼저 분석 결과 제시
- 역량 갭 분석 리포트 출력
- 1/3/5년 목표 로드맵 부여
- 컨설팅식 해결책 제안
- "당신은 이런 유형입니다" 라벨링

---

## 서비스 경험의 핵심

**다른 앱**
```
설문 → 분석 → 결과 제시
(사용자를 피험자로)
```

**이 서비스**
```
최소 설문 → 강점 정보 확보 → 코칭 대화 → 발견 → 사용자가 직접 정리
(사용자를 탐험가로)
```

사용자가 세션을 마쳤을 때 느끼는 것:
"대화하다 보니 내가 어떤 사람인지 알게 됐어."

## 팀의 차별화 자산

- **갤럽 코칭 자격증 보유자 2명** — 34개 키워드를 대화로 역설계하는 능력
- **HR 출신 2명** — 직장인의 커리어 고민 패턴에 대한 깊은 이해
- **MCC 기준 시스템 프롬프트** — AI가 진짜 코칭을 구현할 수 있는 설계

다른 팀은 이 설계를 할 수 없다.
